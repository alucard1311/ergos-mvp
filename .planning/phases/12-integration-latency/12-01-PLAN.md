<task type="auto">
  <name>Task 1: Create Pipeline orchestration module</name>
  <files>src/ergos/pipeline.py</files>
  <action>
    Create `src/ergos/pipeline.py` that:

    1. Define `Pipeline` dataclass containing all components:
       - config: Config
       - state_machine: ConversationStateMachine
       - vad_processor: VADProcessor
       - stt_processor: STTProcessor
       - llm_processor: LLMProcessor
       - tts_processor: TTSProcessor
       - connection_manager: ConnectionManager
       - data_handler: DataChannelHandler
       - app: web.Application (aiohttp signaling app)

    2. Create `async def create_pipeline(config: Config) -> Pipeline`:
       a. Instantiate state machine (ConversationStateMachine)
       b. Instantiate VAD processor (VADProcessor with state machine callbacks)
       c. Instantiate STT components:
          - WhisperTranscriber with config.stt settings
          - STTProcessor with transcriber
       d. Instantiate LLM components:
          - Load persona from config.persona_file using load_persona()
          - LLMGenerator with config.llm settings
          - LLMProcessor with generator and persona.system_prompt
       e. Instantiate TTS components:
          - TTSSynthesizer with config.tts settings
          - TTSProcessor with synthesizer
       f. Create connection manager (ConnectionManager)
       g. Create data channel handler (DataChannelHandler with vad_processor, state_machine)

    3. Wire callbacks:
       - Register data_handler.get_state_callback() with state_machine.add_listener()
       - Register stt_processor as VAD event listener (vad_processor.add_callback)
       - Register llm_processor.process_transcription as STT callback
       - Register tts_processor.receive_token as LLM token callback
       - Create async audio output callback that pushes samples to all TTSAudioTracks via connection_manager
       - Register audio output callback with tts_processor
       - Register tts_processor.clear_buffer as barge-in callback on state_machine

    4. Create on_incoming_audio callback:
       - Convert incoming samples to AudioChunk
       - Feed to stt_processor.on_audio_chunk
       - This bridges WebRTC incoming audio to STT pipeline

    5. Create signaling app with create_signaling_app():
       - Pass connection_manager
       - Pass data_handler
       - Pass on_incoming_audio callback

    6. Return Pipeline with all components wired

    Note: Models load lazily on first use (already implemented in each component)
  </action>
  <verify>python -c "from ergos.pipeline import create_pipeline" succeeds</verify>
  <done>Pipeline module creates and wires all components</done>

       a. Create pipeline: self._pipeline = await create_pipeline(self.config)

       h. Call stop()
