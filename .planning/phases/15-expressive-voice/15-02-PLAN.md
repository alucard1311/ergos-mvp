<objective>
Add emotion markup preprocessing that transforms LLM output into expressive Orpheus TTS input with emotion tags and sarcastic timing pauses.

Purpose: Orpheus TTS supports inline emotion tags (<laugh>, <sigh>, <chuckle>, etc.) but the LLM does not naturally emit these. This plan bridges the gap: (1) the LLM system prompt is updated to emit lightweight emotion hints (e.g., *laughs*, *sighs*, ...) when appropriate, and (2) an EmotionMarkupProcessor converts these hints plus detected patterns (questions, exclamations, sarcasm cues) into Orpheus tags and strategic pauses. This delivers the perceptible expressiveness required by VOICE-04.

Note on commands/imperative sentences: Orpheus 3B renders imperative sentences ("Turn off the lights", "Open the file") with natural emphasis natively -- the model's prosody handles commanding tone without any special markup, same as it handles question intonation naturally. No EmotionMarkupProcessor intervention is needed for commands.

Output: EmotionMarkupProcessor class, updated system prompt for emotion hints, TTSProcessor integration, pipeline.py engine wiring, and unit tests. Concludes with human-verify checkpoint.
</objective>
