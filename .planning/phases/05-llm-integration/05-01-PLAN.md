    1. Create src/ergos/llm/generator.py with LLMGenerator class:

       - __init__(self, model_path: str, n_ctx: int = 2048, n_gpu_layers: int = -1):
         - Store config but don't load model yet (lazy load like WhisperTranscriber)
         - n_ctx: Context window size (2048 for Phi-3 Mini is safe)
         - n_gpu_layers: -1 means use all GPU layers if available

       - _ensure_model(self) -> Llama:
         - Lazy load model on first use
         - Log model loading (this can take time)
         - Cache in self._model
         - Use n_threads=4 for CPU inference

       - generate(self, prompt: str, config: GenerationConfig = None) -> CompletionResult:
         - Build full prompt (will add system prompt in processor)
         - Call model.create_completion() with config params
         - Return CompletionResult with text and token counts

       - generate_stream(self, prompt: str, config: GenerationConfig = None) -> AsyncIterator[str]:
         - Same as generate but yields tokens as they're generated
         - Use model.create_completion(stream=True)
         - Run in thread pool to not block event loop

       - @property model_loaded(self) -> bool
       - @property context_size(self) -> int
       - close(self): release model resources

    2. Update src/ergos/llm/__init__.py to export LLMGenerator

    IMPORTANT: llama-cpp-python Llama() is synchronous. Wrap calls in
    asyncio.run_in_executor() like STT does for transcription.
