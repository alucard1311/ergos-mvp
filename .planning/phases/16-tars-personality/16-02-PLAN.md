**2. src/ergos/memory/store.py (NEW)**
Following the kitchen plugin pattern from src/ergos/plugins/kitchen/memory.py:

- `MEMORY_PATH = Path.home() / ".ergos" / "memory.json"`
- `MEMORY_BUDGET = 15` (max entries injected per session)
- `MEMORY_MAX_STORED = 100` (max entries before pruning)

- `EXTRACTION_PROMPT` — template string per RESEARCH.md Pattern 3:
  "Review this conversation and extract facts worth remembering about the user.
  Conversation: {history}
  Extract 0-5 memorable items. For each, write one clear sentence and categorize it.
  Categories: preference (likes/dislikes/habits), fact (personal info), moment (notable exchange or joke).
  Only extract genuinely useful or memorable information — skip trivial exchanges.
  Format: CATEGORY: sentence
  If nothing memorable occurred, respond with: NOTHING"

- `class MemoryStore`:
  - `__init__(self, storage_path: Path | None = None)`: Uses storage_path or MEMORY_PATH
  - `load(self) -> list[MemoryEntry]`: Read JSON file, parse list of dicts into MemoryEntry objects. Return [] if file missing. Use try/except with logging on parse errors.
  - `save(self, entries: list[MemoryEntry]) -> None`: Serialize via dataclasses.asdict(), write with json.dumps(indent=2). Create parent dirs with mkdir(parents=True, exist_ok=True).
  - `get_budget(self, entries: list[MemoryEntry], n: int = MEMORY_BUDGET) -> list[MemoryEntry]`: Sort by timestamp descending, take first n, increment access_count on each returned entry. Return the selected entries.
  - `prune(self, entries: list[MemoryEntry], max_size: int = MEMORY_MAX_STORED) -> list[MemoryEntry]`: If len(entries) <= max_size, return as-is. Otherwise compute score for each entry: normalize timestamps to 0-1 range, normalize access_counts to 0-1 range, score = 0.7 * norm_timestamp + 0.3 * norm_access_count. Sort by score descending, keep top max_size.

- `parse_extraction_result(text: str) -> list[MemoryEntry]`:
  If text.strip().upper() == "NOTHING", return [].
  Split by newlines, for each non-empty line try to split on first ": " (colon-space).
  Left part is category (lowercased, stripped). Right part is content.
  Valid categories: {"preference", "fact", "moment"}. Skip lines that don't match.
  Return list of MemoryEntry objects.

- `format_history_for_extraction(messages) -> str | None`:
  Takes list of Message-like objects (with .role and .content attributes).
  If len(messages) < 4, return None (skip extraction per Pitfall 6).
  Take last 20 messages. Format as "USER: content" or "ASSISTANT: content" per line.
  Return joined string.
