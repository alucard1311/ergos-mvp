Per CONTEXT.md locked decision: "Short VAD end-of-speech threshold (~500ms) for fast response."

Change `redemptionFrames: 45` to `redemptionFrames: 16`:
```dart
redemptionFrames: 16, // ~512ms pause before speech_end (16 * 32ms) — reduced from 45 (~1.4s) for faster turn-taking
```

This makes VAD fire `speech_end` after ~512ms of silence instead of ~1.4s, which is critical for sub-300ms response time (VOICE-01). The total latency chain is: user stops speaking -> 512ms VAD -> 0ms STT/LLM start -> ~300ms first audio. Without this change, the chain adds 1.4s of unnecessary waiting.

Note: This change may cause more false positives for brief pauses in speech. The `minSpeechFrames: 5` (~160ms) setting helps filter very short noise, and `positiveSpeechThreshold: 0.6` is conservative enough to avoid noise triggers. If users report issues with natural pauses being interpreted as speech_end, this can be tuned later.
