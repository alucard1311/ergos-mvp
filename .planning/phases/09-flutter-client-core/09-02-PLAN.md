    Create client/lib/services/webrtc_service.dart implementing RESEARCH.md patterns:

    Class WebRTCService with:
    - Private fields: _pc (RTCPeerConnection?), _dataChannel (RTCDataChannel?), _localStream (MediaStream?)
    - Callbacks: onConnectionStateChanged, onServerStateChanged, onDataChannelReady
    - Constructor taking SignalingService

    Method `Future<void> connect()`:
    1. Request microphone permission using requestMicrophonePermission()
    2. Get local audio stream: navigator.mediaDevices.getUserMedia({'audio': true, 'video': false})
    3. Create peer connection with STUN: {'iceServers': [{'urls': 'stun:stun.l.google.com:19302'}]}
    4. Set up onConnectionState callback to track connection state
    5. Set up onTrack callback for incoming server audio (just log "Received server audio")
    6. Add local audio tracks to peer connection
    7. Create data channel BEFORE offer: createDataChannel('data', RTCDataChannelInit()..ordered = true)
    8. Set up data channel onMessage handler to parse JSON and emit ServerState
    9. Set up data channel onDataChannelState to emit when ready
    10. Create offer with offerToReceiveAudio: true
    11. Set local description
    12. Send offer via SignalingService, get answer
    13. Set remote description with answer

    Method `void sendDataChannelMessage(Map<String, dynamic> message)`:
    - Check data channel state is open
    - Send RTCDataChannelMessage(jsonEncode(message))

    Method `Future<void> disconnect()`:
    - Close data channel
    - Close peer connection
    - Dispose local stream

    CRITICAL: createDataChannel MUST be called BEFORE createOffer (pitfall #1 from RESEARCH.md)
  </action>
